## Cursor

初体验 Cursor 是非常惊艳，我格外喜欢它的 Tab 能力；其补全是**快速且准确**的。

+ Tab 的爽点：
    + 当我想好怎么写，并键入一部分字符后，Tab 就能尝试补全，会增删改若干行的内容，我对此的接受率非常高；结果就是我大概键入5%-10%的字符，剩下由 Tab 补全，这是提效。
    + 于是编码过程就变成了"想好怎么写->键入部分字符->出现补全->review补全->接受补全"的循环；
        + review的过程是包含在循环里面的，所以没有明显的需要review大量代码的体感。
        + 虽然大部分代码都是AI生成的，但是我对代码细节是了解的（仿佛全是我写的），而不是全量生成完再review，后者对代码细节体感没有那么强烈（因为完全不是我写的）。

随着AI生成的代码质量越来越高，速度越来越快，AI编程的形式更多了，此时人成为了整个链路的效率瓶颈。

+ 对于一个功能/需求可能会plan出多个独立的部分，然后分别建立 sub agent 生成，此时的效率肯定比上述循环要高。
+ 可以有human no in loop的迭代循环，比如需求发现和验证，代码生成，代码review和测试，分别由 sub agent 组成，通过"人类不在循环中"的循环，就可以完成产品/需求的迭代，此时的效率肯定比上述循环要高。
> 以上是我目前能想到的未来的开发场景
