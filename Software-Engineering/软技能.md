## 毕业工作后，如何学习进步？

+ 上学时如何学习？
    + 计算机科学基础科目——大部头著作&国外公开课&论文；
    + 实践——项目&实习

+ 为什么上学这样学是有效的？
    + 企业对校招生的要求就是基础牢&快上手，上面是“对症下药”的。
    + 对于一个领域的知识，学生往往是什么都不懂了，此时对其的学习&实践，是“见效”飞快的。

+ 为什么工作之后，同样的学习方式就没有那么推荐了？
    + 基础科目已经进入比较复杂的阶段，此时边际效应递减，学到的知识在工作上使用上的概率小。

+ 沟通的正确姿势：
    + 背景：需求涉及多个系统，对整体链路设计只有模糊的感觉，需要和各方对齐：存量逻辑和链路是什么样子的，从而设计增量逻辑和链路。
    + 动作：
        + 错误的动作：才介入就拉全体会
            + 全体会的成本是比较高的，所有人都在时间不多，尽量在必须的时候才拉。
            + 作为会议发起人的 RDO 没有会议主线，会导致会议低效。
            
            上述行为会显得 RDO 很不专业。
        + 正确的动作：对于链路模糊的地方，找到其相关方（一到两个人），线下/线上小范围沟通，通过这种方式理清整体逻辑、链路或者其他完成需求的信息。最终你会发现根本不需要全体的会议。
